Signaling Gateway in a Next-Generation Services Application

Overview

This application supports Intelligent Network (IN) services for subscribers of both IP networks and the PSTN, regardless of the location of the application server or database. For example, TCAP queries and responses are used to access database(s) on IP networks to assist local telephone switches in offloading congestion caused by Internet call access. The signaling gateways are also used by applications that need to access PSTN databases. For example, Internet Call Waiting applications may need to access the CNAM database for calling name information. Other applications that might require a signaling gateway to interwork database queries and responses between PSTN and IP networks include calling card verification and mobile telephone roaming.

Next-Generation Services

  • Distributed database application architecture on IP side
  • Single point code (SEPs) or multiple point code (STPs) applications
  • Uses TCAP for AIN, mobile and other SCP applications

 

What products do we provide in this application?

SEGway™ Signaling Gateways: The SEGway Signaling Gateway is an ideal product for distributed IN services. The distributed SS7 stack architecture gives solution providers a cost effective way to deploy new services in stages. For example, service providers can start with a smaller test market, and then as the service demand increases or is proven, they can incrementally add gateways to grow the capacity. Furthermore, onboard protocol conversion enables the service to be deployed in different international markets without concern for the variations in the local version of the SS7 protocol.
